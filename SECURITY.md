# Security Policy

## Supported versions

No stable version is currently supported. This repository is pre-alpha.

## Reporting

Do not publish credentials, private datasets, personal information, or exploitable security details in a public issue. Use GitHub private vulnerability reporting when enabled.

## Project-specific risks

Security reviews should consider:

- malicious or misleading semantic contracts;
- unauthorized promotion of inferred mappings;
- unsafe external references;
- schema denial-of-service inputs;
- sensitive provenance leakage;
- policy declarations being mistaken for policy enforcement;
- model prompt injection embedded in source metadata.
