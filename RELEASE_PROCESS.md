# Release Process

A release is permitted only when:

1. all required checks pass;
2. official fixtures are frozen for the release;
3. specification and schema versions agree;
4. changelog and migration notes are complete;
5. conformance limitations are explicit;
6. project status and next action are updated;
7. the release commit is tagged;
8. generated artifacts match the tagged source.

No release may claim industry-standard status without documented independent adoption.
