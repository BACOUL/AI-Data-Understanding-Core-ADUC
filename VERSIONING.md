# Versioning

The project uses `MAJOR.MINOR.PATCH` with prerelease identifiers.

During experimentation:

```text
0.1.0-alpha.0
0.1.0-alpha.1
0.1.0-beta.0
0.1.0
```

A release must identify separately:

- specification version;
- schema version;
- reference-tool version;
- conformance-suite version.

Breaking normative changes require a compatibility note even before `1.0.0`.
