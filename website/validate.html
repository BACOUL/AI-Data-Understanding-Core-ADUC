One command. A complete Core validation pipeline.

The unified validator loads JSON, applies 14 local Draft 2020-12 schemas, checks Core architecture and runs applicable profile evaluators.

Official CLI

Validate a complete contract.

No hosted upload or remote schema resolution is required.

python tools/aduc_core.py validate examples/core/complete-model.example.json
python tools/aduc_core.py validate examples/core/complete-model.example.json --format json
python tools/aduc_core.py validate contract.json --schema-only
python tools/aduc_core.py validate contract.json --no-profile-evaluation

Validation order

Every layer answers a different question.

  1. 01
    Load JSON

    Is the input readable, bounded and free of duplicate members?

  2. 02
    Apply schemas

    Do the Core and module shapes conform to the 14 local Draft 2020-12 schemas?

  3. 03
    Check architecture

    Do identifiers, references, bindings, ownership rules and graph limits hold?

  4. 04
    Evaluate profiles

    Do applicable unit, time, identity, provenance, uncertainty, relation and policy rules pass?

  5. 05
    Return final status

    Valid, valid with warnings, requires human review or blocked.

Stable report shape

Diagnostics stay attributable.

Schema, architecture and profile outcomes remain separate so a consumer can see what failed and why.

{{
  "reportVersion": "0.1.0",
  "valid": false,
  "outcome": "blocked",
  "contractId": "urn:example:contract:1",
  "summary": {{"errors": 1, "warnings": 0, "humanReview": 0}},
  "pipeline": {{
    "jsonLoaded": true,
    "schemaValid": true,
    "architectureValid": false,
    "profileEvaluation": true
  }},
  "diagnostics": []
}}

Validation boundaries

A valid contract is structured and internally consistent — not automatically true or legally permitted.

No factual truth proofNo automatic authorityNo universal permissionNo unapproved remote trust
