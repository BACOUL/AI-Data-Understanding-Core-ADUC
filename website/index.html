Experimental open standard

A portable understanding layer for data and AI.

ADUC is a model-independent contract that lets data describe its structure, meaning, context, provenance, uncertainty, relations and conditions of use to AI systems.

Read the Core draft View on GitHub

AI Data Understanding Core is a provisional working name. ADUC is not yet a recognized standard, and full multi-model interoperability is not yet proven.

The problem

Readable syntax is not shared understanding.

A model can parse {"flow": 118}, but it cannot know with certainty what the field means, which unit applies, when the observation occurred, who produced it, how reliable it is, or whether it may be compared with another source.

01

Meaning is rebuilt repeatedly

Organizations encode the same data meaning again in prompts, ETL jobs, connectors, dictionaries and application code.

02

Models receive different explanations

Each provider or application can end up with its own semantic mapping, producing inconsistent interpretation and hidden coupling.

03

Uncertainty disappears

Inferred, reviewed, canonical and contested interpretations are often flattened into one undocumented answer.

How it works

Describe once. Validate. Reuse.

The original data remains unchanged. A portable ADUC contract accompanies it and exposes the meaning that compatible AI systems and applications need.

Original dataJSON, CSV and later additional resource types.
ADUC contractStructure, semantics, identity, context and evidence.
ValidationSchema, lifecycle, uncertainty and conflict checks.
Shared useAI systems, agents, analytics and applications.

Current status

A real technical foundation, not a finished standard.

The repository already contains tested components, but the complete Core schema, compiler and full interoperability proof remain future gates.

Design principles

Open, explicit and provider-independent.

ADUC is useful only if it remains understandable without a proprietary service and preserves uncertainty instead of hiding it.

↗

Reuse standards

JSON-LD, RDF, Croissant, PROV-O, DQV, ODRL, JSON Schema and established vocabularies remain foundations.

?

Preserve uncertainty

Unknown, inferred, reviewed, canonical and contested information cannot be silently treated as equivalent.

∞

Avoid vendor lock-in

The contract and conformance rules must remain usable by independent models, tools and organizations.

The target

Make existing data portable between an organization’s AI systems.

The open standard remains free. Commercial adoption tools may later provide hosted compilation, collaborative review, private registries, enterprise connectors, audits and support.
