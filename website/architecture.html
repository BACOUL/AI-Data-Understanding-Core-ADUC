Architecture

A small Core with explicit layers around it.

ADUC separates the normative contract, schema enforcement, deterministic evaluation and future adoption tooling. Vision does not masquerade as implementation.

01

Original resource

The source remains authoritative. ADUC binds it through identifiers, versions and digests without replacing its bytes.

02

Normative Core

Ten blocks define responsibility boundaries. The minimum envelope is aduc + resource + structure.

03

Schema family

Fourteen local Draft 2020-12 schemas enforce module shapes and closed Core objects.

04

Unified validator and comparator

One local interface orchestrates schema, architecture and applicable domain rules with deterministic reports.

05

Legacy semantic profile migration

The accepted migration tool converts the earlier standalone profile only when an explicit manifest supplies missing Core facts.

06

Future adoption layer

Compiler, review UI, packages, registry and MCP adapter remain separate roadmap work.

Provider neutrality

The Core does not depend on a model vendor.

Contracts are local JSON documents. Validation and comparison are deterministic Python tools. A consumer can be an AI system, an agent, an application or a human review workflow.
