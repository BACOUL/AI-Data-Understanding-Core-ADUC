Try ADUC in 5 minutes

The current reference tools exercise accepted Core behavior for semantic mappings, source binding, units, time, identity, provenance, uncertainty, relations and policy. The unified full-Core validator and compiler remain planned work.

Important scope: these commands exercise separate accepted reference profiles. They do not yet validate the complete ten-block contract shown on the Example page.

1. Requirements

  • Git
  • Python 3.12 or a compatible recent Python 3 version
  • Internet access only for the initial dependency installation

2. Clone the repository

Terminal
git clone https://github.com/BACOUL/AI-Data-Understanding-Core-ADUC.git
cd AI-Data-Understanding-Core-ADUC

3. Install development dependencies

Terminal
python -m pip install -r requirements-dev.txt

4. Run the complete local checks

Terminal
python tools/validate_contracts.py
python -m unittest discover -s tests/validator -p "test_*.py"
python -m unittest discover -s tests/comparator -p "test_*.py"
python -m unittest discover -s tests/jsonld -p "test_*.py"
python -m unittest discover -s tests/conformance -p "test_*.py"
python -m unittest discover -s tests/epistemic -p "test_*.py"
python -m unittest discover -s tests/source_binding -p "test_*.py"
python -m unittest discover -s tests/units -p "test_*.py"
python -m unittest discover -s tests/time -p "test_*.py"
python -m unittest discover -s tests/identity -p "test_*.py"
python -m unittest discover -s tests/provenance -p "test_*.py"
python -m unittest discover -s tests/uncertainty -p "test_*.py"
python -m unittest discover -s tests/relations -p "test_*.py"
python -m unittest discover -s tests/policy -p "test_*.py"

The GitHub Actions workflow remains the source of truth for the complete check set.

5. Validate one semantic-mapping profile

Text report
python tools/aduc_validate.py \
  examples/authoring/river/reviewed.aduc.json

Machine-readable report:

JSON report
python tools/aduc_validate.py \
  examples/authoring/river/reviewed.aduc.json \
  --format json

6. Declare a local trusted authority

A canonical mapping names an authority. Local trust configuration can suppress an unverified-authority warning, but it is not cryptographic proof or global trust.

Terminal
python tools/aduc_validate.py \
  examples/authoring/machine/canonical.aduc.json \
  --trusted-authority https://factory.example/id/data-authority

7. Compare two profiles

Terminal
python tools/aduc_compare.py \
  examples/comparison/fr/profile.aduc.json \
  examples/comparison/us/profile.aduc.json \
  --trusted-authority-b https://example.org/id/us-data-authority

The comparator uses only published semantic targets and mapping relations. It does not guess unit conversions, temporal alignment or entity identity when those dimensions are absent.

8. Evaluate policy and permitted-use cases

Policy harness
python tools/aduc_policy.py \
  examples/policy/reference-cases.json \
  examples/policy/invalid-cases.json

The evaluator reproduces deterministic permit, deny, notApplicable, indeterminate and requiresHumanReview outcomes. These outcomes are not legal advice and do not grant access.

9. Expand and normalize a profile as RDF

Normalized N-Quads
python tools/aduc_rdf.py \
  examples/authoring/river/reviewed.aduc.json

Other formats:

Expanded or compacted JSON-LD
python tools/aduc_rdf.py profile.aduc.json --format expanded
python tools/aduc_rdf.py profile.aduc.json --format compacted

10. Verify the frozen conformance package

Terminal
python tools/aduc_conformance.py verify-package \
  examples/conformance/package

The committed example model results are illustrative. They must not be counted as proof of interoperability between two external AI providers.

11. What to build next

For specification contributors

  • freeze the normative Core object model;
  • define object ownership and modular boundaries;
  • update the complete example;
  • challenge duplicated or unnecessary properties.

For implementers

  • review the accepted profile tools;
  • do not build the compiler before the Core model and schema;
  • prepare modular schema feedback;
  • run the frozen package with independent consumers only after the full-Core package exists.

Current limitations

Do not claim yet: complete Core conformance, automatic JSON/CSV compilation, production legal-policy enforcement, a production semantic registry, or proven interoperability between two qualifying external models.
