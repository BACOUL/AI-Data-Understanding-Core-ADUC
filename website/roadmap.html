Official roadmap

The project follows a strict construction order. Later extensions and the anticipation engine cannot replace the Core, schema, tools and interoperability proof that must come first.

Open the master planTrack issues
Mandatory order: Core model → Schema family → Unified validator → Complete examples → JSON/CSV compiler → Multi-model demonstration → Extensions → Anticipation engine.
0Complete

Definition

Mission, non-goals, nine domain profiles, normative Core object model, module boundaries, extension rules and one complete ten-block example.

  • The minimum envelope is aduc + resource + structure
  • Every normative fact has one owning module
  • Twenty-five architecture counterexamples are checked in CI
1Next active task

Standard v0.1 schema family

Draft 2020-12 root and modular schemas, qualification and extension definitions, ten valid complete examples and ten invalid complete examples.

  • The object model and dependency graph are frozen
  • The existing root schema is only a bootstrap and must be replaced by the official family
2Partial foundation

Reference implementation

Unified full-Core validator, stable error catalogue, comparator, SDKs, conformance suite and updated “Try in 5 minutes” guide.

  • Separate profile tools already exist
  • Unified Core migration remains pending
3Not started

JSON/CSV compiler

Import data, inspect structure, propose concepts and units, detect ambiguity and export an inferred contract.

4Not started

Review interface

Show unknown fields, low-confidence mappings, ambiguous units, time conflicts, identity candidates, provenance gaps, uncertainty assumptions and relation or policy conflicts.

5Harness ready

Value and interoperability demonstration

Manual and assisted mapping benchmarks, two incompatible sources, two independent AI consumers, one frozen package and one reproducible comparison report.

  • Provider-neutral harness exists
  • A value benchmark and qualifying external runs remain absent
6Blocked by Core proof

First extension

Choose Dataset or Live Data only after the Core passes the value and interoperability gate.

7Future

Situation & Action extension

Represent situations, possible developments, actions, constraints and evidence without turning the Core into a decision engine.

8Future application

Anticipation engine

A separately governed application using ADUC Core plus Live Data and Situation & Action extensions.

9Future ecosystem

Registry, connectors and governance

Public and private registries, community extensions, certification, professional support and independent governance.

Version 0.1

Completion requires reproducible evidence.

The project will not call v0.1 complete because a website or a long document exists. Each practical deliverable must be present and independently testable.
