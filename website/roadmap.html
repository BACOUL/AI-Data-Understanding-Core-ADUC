Official roadmap

The project follows a strict construction order. Later extensions and the anticipation engine cannot replace the Core, schema, tools and interoperability proof that must come first.

Open the master planTrack issues
Mandatory order: Core → Schema → Validator → Examples → JSON/CSV compiler → Multi-model demonstration → Extensions → Anticipation engine.
0In progress

Definition

Mission, non-goals, full Core draft, official structure, master plan, decisions and one complete example.

  • Epistemic, source-binding, unit, temporal, identity, provenance, uncertainty, quality, relation and policy profiles are reference-tested
  • The normative Core object model and modular boundaries are the next active decision
  • The official schema family remains after the object model is frozen
1Profile foundation

Standard v0.1

Normative Core blocks, schema family, vocabulary, versioning, ten valid examples and ten invalid examples.

  • Nine foundational Core profiles now have deterministic evaluators
  • The complete Core object model and schema do not exist yet
2Partial foundation

Reference implementation

Complete validator, TypeScript SDK, Python SDK, comparator, conformance suite and “Try in 5 minutes” guide.

  • Mapping, source, unit, time, identity, provenance, uncertainty, relation and policy tools exist
  • Full-Core migration remains pending
3Not started

JSON/CSV compiler

Import data, inspect structure, propose concepts and units, detect ambiguity and export an inferred contract.

4Not started

Review interface

Show unknown fields, low-confidence mappings, ambiguous units, time conflicts, identity candidates, provenance gaps, uncertainty assumptions and relation or policy conflicts.

5Harness ready

Value and interoperability demonstration

Manual and assisted mapping benchmarks, two incompatible sources, two independent AI consumers, one frozen package and one reproducible comparison report.

  • Provider-neutral harness exists
  • A unified full-Core package, value benchmark and external qualifying runs remain absent
6Blocked by Core proof

First extension

Choose Dataset or Live Data only after the Core passes the value and interoperability gate.

7Future

Situation & Action extension

Represent situations, possible developments, actions, constraints and evidence without turning the Core into a decision engine.

8Future application

Anticipation engine

A separately governed application using ADUC Core plus Live Data and Situation & Action extensions.

9Future ecosystem

Registry, connectors and governance

Public and private registries, community extensions, certification, professional support and independent governance.

Version 0.1

Twelve completion gates.

The project will not call v0.1 complete because a website or a long document exists. Each practical deliverable must be present and reproducible.

Immediate next milestone

Freeze the normative Core object model.

The next work must define object ownership, cardinalities, deterministic references, modular boundaries, versioning, extensions and migration so the official schema family can be implemented without new architectural guesses.
