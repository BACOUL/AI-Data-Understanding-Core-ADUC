Raw data → portable understanding

River station R42

A four-field JSON record is easy to parse but incomplete to interpret. The accompanying ADUC contract adds structure, meaning, identity, time, provenance, uncertainty, relations and usage conditions.

Open example filesRead the specification
Draft limitation: this is the first informative full-Core example. The final aduc-core.schema.json has not yet been implemented, so the example must not be described as fully validated Core conformance.
river-r42.data.json
{
  "station": "R42",
  "flow": 118,
  "quality": 2,
  "timestamp": "2026-07-13T12:00:00Z"
}

What is missing?

  • What does flow mean?
  • Which unit applies?
  • What does quality code 2 mean?
  • Is the timestamp observation time or ingestion time?
  • What identity system defines R42?
  • Who produced the measurement?
  • What uncertainty applies?
  • Which uses are permitted?
The contract

Meaning attached without rewriting the source.

The complete example uses all ten candidate Core blocks. The extract below focuses on the field semantics and context; the repository file also includes resource binding, identity, provenance, uncertainty, relations and policy.

river-r42.aduc.json — selected extract
{
  "aduc": {
    "version": "0.1.0-draft",
    "contractId": "urn:aduc:contract:river-r42:2026-07-13",
    "status": "reviewed"
  },
  "semantics": {
    "fields": {
      "flow": {
        "concept": "https://example.org/vocab/env/WaterDischarge",
        "unit": "https://qudt.org/vocab/unit/M3-PER-SEC",
        "mappingRelation": "http://www.w3.org/2004/02/skos/core#exactMatch",
        "mappingStatus": "reviewed",
        "confidence": 0.96,
        "confidenceMethod": "urn:aduc:method:human-review-v1"
      },
      "quality": {
        "concept": "https://example.org/vocab/quality/ObservationQualityCode",
        "valueMap": {
          "2": "https://example.org/vocab/quality/Provisional"
        },
        "mappingStatus": "canonical"
      }
    }
  },
  "context": {
    "timeField": "timestamp",
    "timeMeaning": "observationTime",
    "timezone": "UTC",
    "spatialCoverage": "France"
  },
  "uncertainty": {
    "measurements": [{
      "appliesTo": "flow",
      "relativeError": 0.04,
      "confidenceLevel": 0.95
    }]
  }
}
Expected interpretation

What a compatible consumer should preserve.

ADUC does not simply produce a prettier data dictionary. It defines portable claims and the confidence or authority attached to each claim.

≈

Semantic meaning

flow represents water discharge measured in cubic metres per second.

2

Code meaning

quality = 2 means the observation is provisional, according to the source authority’s value map.

◷

Time meaning

timestamp is observation time in UTC, not merely an undocumented date string.

ID

Identity

R42 uses the declared station identifier scheme and represents a hydrometric station.

±

Uncertainty

The flow measurement carries a declared relative error of 4% at a 95% confidence level.

§

Usage conditions

The example declares public classification and intended uses such as research and flood-risk analysis.

Why status matters

Reviewed is not canonical.

The flow mapping is reviewed by a hydrologist, while the quality-code mapping is canonical because it is published by the source authority. A compatible AI must preserve that difference.

Reviewed claim: strong human review evidence, but not necessarily published by the source owner.
Canonical claim: published by the declared source authority, but still dependent on authority verification and conflict handling.
