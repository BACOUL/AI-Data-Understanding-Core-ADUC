# Success Criteria

## v0.1-alpha acceptance criteria

All conditions are required:

1. The Core specification is readable in under 30 minutes.
2. The JSON Schema validates every official valid fixture and rejects every official invalid fixture.
3. A developer can author a basic contract from the documentation without project-maintainer assistance.
4. Two incompatible data sources can be aligned at concept, unit, time, and entity-candidate levels.
5. Inferred mappings remain distinguishable from reviewed, verified, and canonical mappings.
6. At least two independent model providers consume the same contracts.
7. Results are compared using a public, repeatable evaluation protocol.
8. No provider-specific field is required by the Core.
9. Every normative change is traceable to an issue, ADR, pull request, and changelog entry.
10. The public claim remains limited to what the tests demonstrate.
