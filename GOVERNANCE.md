# Governance

## Bootstrap phase

During the bootstrap phase, the project founder is the final maintainer. Decisions must nevertheless be documented publicly in issues, ADRs, and pull requests.

## Decision classes

- Editorial: documentation clarification with no normative change
- Implementation: reference tooling with no specification change
- Normative: changes conformance or contract meaning
- Architectural: changes Core boundaries, compatibility, governance, or extension model

Normative and architectural changes require an issue, ADR, tests, compatibility analysis, and explicit approval.

## Future transition

After independent adopters exist, governance should move toward a multi-maintainer technical committee with documented voting, appeals, and conflict-of-interest rules.
